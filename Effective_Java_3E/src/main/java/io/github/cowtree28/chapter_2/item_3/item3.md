# 3. private 생성자나 열거 타입으로 싱글턴임을 보증하라.
> 싱글턴을 써야하는 경우 열거타입, private 생성자를 사용하자.

## 싱글턴이란?
인스턴스를 오직 하나만 생성할수 있는 클래스를 말함.

### 사용처
- 무상태 객체
- 시스템 컴포넌트

### 특징
이러한 클래스를 싱글턴으로 만들면 테스트하기 어렵다.  
싱글턴 인스턴스를 가짜 구현으로 대체하기 어렵기 때문

### 싱글턴을 만드는 방법
#### 1. public static 멤버가 public인 방법
생성자는 INSTANCE 필드를 초기화할때 한번만 호출된다.  
이때 리플렉션 API(AccessibleObject.setAccessible)을 사용해 private 생성자를 호출할수 있는데,  
이는 생성자를 수정해 두번째 객체가 생성될때 예외를 던져 예방할수 있다.

#### 2. 정적 팩터리 메서드를 public으로 만드는 방법
위와 같지만 생성자를 정적팩터리 메서드로 변경할수 있다.  
리플렉션의 위험도도 똑같이 적용된다.  
또한 싱글턴임을 숨길수 있다. 따라서 원할때 마음대로 싱글턴이 아니게 변경할수 있다.

#### 3. 열거타입의 싱글턴 선언
public 방식보다 간결하고, 리플렉션도 방지 가능하다.  
하지만 상속을 해야하는 경우라면 사용할 수 없다.
