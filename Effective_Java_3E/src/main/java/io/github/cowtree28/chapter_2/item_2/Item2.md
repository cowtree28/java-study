# 2. 생성자에 매개변수가 많다면 빌더를 고려하라
> 매개변수가 많으면 생성자를 활용하기 어려우니 Builder를 활용하자

## Builder를 쓰는 이유
기존에는 생성자에 여러 정보가 들어가 매개변수가 많을때 대응하기 어렵다.  
그래서 점층적 생성자 ( **매개변수의 개수를 하나씩 늘려가는것** ) 을 활용하였지만 역시나 작성이 힘들다.  
그래서 setter를 활용한다고 해도 객체의 완성을 위해 여러개의 메서드 호출이 필요하다.  
이때 Builder패턴을 사용한다.

## Builder란?
객체를 직접 만드는 대신 필수 매개변수로 생성자를 호출해 값을 set하고,  
이를 build메서드를 통해 필요한 객체를 생성한다.
### 특징
- 가변인수 매개변수를 여러개 사용할수 있다.
- 매개변수들을 하나의 필드로 모을 수 있다.
- 상당히 유연하여 한가지 종류의 객체만 반환하지 않고 어떤 필드는 빌더가 알아서 채우도록 할 수 있다.
- 빌더 생성 비용이 크진 않지만 성능에 민감하면 문제가됨
- 매개변수가 4개 이상은 되어야 값어치를 함
- 하지만 생성자, 정적팩터리 방식에서 빌더로 전환할수있지만 빌더로 시작하는편이 낫다고 함
