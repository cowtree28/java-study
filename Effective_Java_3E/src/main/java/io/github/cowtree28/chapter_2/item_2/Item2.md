# 2. 생성자에 매개변수가 많다면 빌더를 고려하라
> 매개변수가 많으면 생성자를 활용하기 어려우니 Builder를 활용하자

## Builder를 쓰는 이유
기존에는 생성자에 여러 정보가 들어가 매개변수가 많을때 대응하기 어렵다.  
그래서 점층적 생성자 ( **매개변수의 개수를 하나씩 늘려가는것** ) 을 활용하였지만 역시나 작성이 힘들다.  
그래서 setter를 활용한다고 해도 객체의 완성을 위해 여러개의 메서드 호출이 필요하다.  
이때 Builder패턴을 사용한다.

## Builder란?
객체를 직접 만드는 대신 필수 매개변수로 생성자를 호출해 값을 set하고,  
이를 build메서드를 통해 필요한 객체를 생성한다.
### 특징
- 이름을 가질수 있다
- 인스턴스를 매번 생성할 필요 X
- 하위 타입 객체 반환 가능
- 매개변수에 따라 다른 클래스 객체 반환 가능
- 반환 객체의 클래스가 존재하지 않아도 된다.(인터페이스 사용)
### 단점
- 정적 팩터리 메서드만 존재하는 경우에 생성자가 없으면 하위 클래스를 만들 수 없다.
- 정적 팩터리 메서드는 프로그래머가 찾기 어렵다.

### 종류
- from: 매개변수 하나 받아서 인스턴스 반환
- of: 매개변수 여러개 받아서 인스턴스 반환
- valueOf: from, of의 자세한버전?
- instance or getInstance: 인스턴스를 반환. 하지만 같은 인스턴스인지, 새 인스턴스인지 보장 X
- create or newInstance: 새 인스턴스를 생성해 반환
- getType: instance 다른 클래스 버전. 하지만 새거인지 보장 X
- newType: instance 다른 클래스 버전. 새거 반환
